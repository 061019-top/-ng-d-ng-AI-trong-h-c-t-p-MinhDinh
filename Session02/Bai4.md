1. Prompt Sai (The Bad Prompt)
Prompt:

"Code của tôi bị lỗi, hãy sửa giúp tôi."

Nhận xét: Prompt này hoàn toàn vô dụng vì thiếu dữ liệu đầu vào (Context). AI không phải "nhà ngoại cảm" để biết cậu đang viết ngôn ngữ gì, đoạn code trông ra sao và thông báo lỗi (Error log) cụ thể là gì.

Gợi ý cải thiện:

"Tôi đang viết code C để giải phương trình bậc 2 nhưng gặp lỗi logic (nhập a=0 chương trình vẫn chạy). Đây là đoạn code của tôi: [Dán code vào đây]. Hãy chỉ ra lỗi sai và giải thích cách sửa."

2. Prompt Đúng 1: Học khái niệm mới (The Concept Learner)
Prompt:

"Đóng vai một giảng viên lập trình C nhiệt tình. Hãy giải thích khái niệm 'Con trỏ' (Pointer) cho sinh viên năm nhất bằng cách sử dụng phép ẩn dụ thực tế (ví dụ: số nhà và địa chỉ). Sau đó, viết một đoạn code ngắn dưới 10 dòng để minh họa sự khác biệt giữa truyền tham trị và truyền tham chiếu."

Nhận xét: Prompt này rất hiệu quả vì thiết lập rõ: Vai trò (Giảng viên), Đối tượng (Sinh viên năm 1), Phương pháp (Ẩn dụ/Analogy) và Yêu cầu cụ thể (Code < 10 dòng). Giúp AI trả về kết quả dễ hiểu, đúng trọng tâm thay vì định nghĩa khô khan.

3. Prompt Đúng 2: Tư duy đa chiều (The Problem Solver)
Prompt:

"Tôi đang học CSS và muốn căn giữa một thẻ div trong trang web. Hãy liệt kê 3 cách khác nhau để làm điều này (ví dụ: Flexbox, Grid, Position). Với mỗi cách, hãy phân tích ưu điểm, nhược điểm và tình huống nên sử dụng. Đừng chỉ đưa code, hãy giải thích tại sao."

Nhận xét: Prompt này xuất sắc ở chỗ nó không chỉ xin đáp án ("How") mà còn yêu cầu tư duy so sánh ("Why/When"). Nó buộc AI phải cung cấp bức tranh toàn cảnh, giúp người học hiểu sâu bản chất vấn đề để chọn giải pháp tối ưu nhất thay vì chỉ copy-paste.
